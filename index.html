Interview Prep

Software Testing Interview Questions

Practice the most commonly asked interview questions to crack your QA job interview with confidence.

Q01 What is the difference between Verification and Validation in Software Testing? ▼
Verification checks whether we are building the product right (process-oriented) — it involves reviews, walkthroughs, and inspections without executing code. Validation checks whether we are building the right product (product-oriented) — it involves actual testing by executing the software to ensure it meets the user requirements.
Q02 What is the difference between Severity and Priority in bug reporting? ▼
Severity is the impact of a bug on the application functionality — how badly the bug affects the system. Priority is the order in which a bug should be fixed — based on business needs. A bug can have high severity but low priority (e.g., spelling mistake on the home page) or low severity but high priority (e.g., logo not displaying on the landing page).
Q03 What is Regression Testing and when do we perform it? ▼
Regression Testing is re-testing the application after a bug fix or new feature is added to ensure that existing functionality has not been broken. We perform regression testing whenever there is a code change, bug fix, or new feature addition in the application.
Q04 What is the difference between findElement() and findElements() in Selenium? ▼
findElement() returns a single WebElement — the first matching element on the page. If no element is found, it throws a NoSuchElementException. findElements() returns a List of WebElements matching the locator. If no elements are found, it returns an empty list (no exception thrown). Use findElements() when you expect multiple elements or want to check if an element exists.
Q05 What is XPath and what are the different types of XPath? ▼
XPath (XML Path Language) is a syntax used to locate elements in an HTML/XML document. Types: (1) Absolute XPath — starts from the root element using / and follows the entire path. It is brittle and not recommended. (2) Relative XPath — starts from anywhere in the document using // and is more flexible and commonly used in Selenium testing.
Q06 What are the advantages of Playwright over Selenium? ▼
Playwright advantages over Selenium: (1) Auto-wait — Playwright automatically waits for elements to be ready before performing actions. (2) Cross-browser — supports Chromium, Firefox and WebKit out of the box. (3) Faster execution. (4) Built-in support for multiple tabs, iframes and shadow DOM. (5) Network interception and mocking. (6) Better debugging with trace viewer. (7) Supports multiple languages — JS, Python, Java, C#.
Q07 How do you handle a new tab or popup in Playwright? ▼
In Playwright, you handle a new tab by listening to the 'page' event on the context before the action that opens the tab: const newPagePromise = context.waitForEvent('page'); await page.locator('a[target="_blank"]').click(); const newPage = await newPagePromise; await newPage.waitForLoadState(); — then you can interact with newPage like any other page object.
Q08 What is the difference between GET and POST HTTP methods? ▼
GET is used to retrieve data from the server. Parameters are sent in the URL. It is idempotent (same request gives same result) and can be cached. POST is used to send/create data on the server. Parameters are sent in the request body. It is not idempotent and is used for creating or updating resources. GET has URL length limitations while POST can send large amounts of data.
Q09 What are common HTTP status codes a tester should know? ▼
Common HTTP status codes: 200 OK (success), 201 Created (resource created), 204 No Content (success with no body), 400 Bad Request (invalid request), 401 Unauthorized (authentication required), 403 Forbidden (no permission), 404 Not Found (resource doesn't exist), 405 Method Not Allowed, 500 Internal Server Error (server-side error), 503 Service Unavailable.
Q10 What is Boundary Value Analysis (BVA) and Equivalence Partitioning (EP)? ▼
Equivalence Partitioning divides input data into partitions where all values in a partition are expected to behave the same way. You test one value from each partition. Boundary Value Analysis tests the boundary values of partitions — the values at the edges (minimum, maximum, just above minimum, just below maximum). BVA is often used together with EP to achieve maximum coverage with minimum test cases.
